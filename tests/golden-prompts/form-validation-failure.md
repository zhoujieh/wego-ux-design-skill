# 表单校验失败回归测试

## 测试目标
验证 Skill 能否生成符合微购规范的表单校验失败 Web 原型项目，包含字段级校验、错误提示、提交失败、恢复等完整链路。

## 输入提示词
使用 $wego-ux-design 生成一个微购表单校验失败 Web 原型项目。用户是需要填写收货地址的买家，需要输入姓名、手机号、详细地址，业务目标是确保地址信息准确完整。

## 期望页面类型
表单型

## 必须命中的组件
- Input
- Button
- Toast

## 必须覆盖状态
- 初始状态
- 姓名为空
- 手机号为空
- 手机号格式错误
- 地址为空
- 地址过短
- 提交中
- 提交失败（服务端校验失败）
- 失败后字段级错误提示
- 修改后可重新提交
- 提交成功

## 必须使用 Token 类别
- color
- typography
- spacing
- radius
- size
- layout
- motion

## 禁止问题
- 禁止输出单文件 HTML
- 禁止内联 style
- 禁止内联 script
- 禁止业务 CSS 硬编码 HEX / px / rgba
- 禁止只静态展示多个状态
- 禁止跳过字段级错误提示
- 禁止跳过失败恢复路径
- 禁止提交失败后不清除错误状态

## 验收清单
- 输出目录包含 index.html、styles/tokens.css、styles/components.css、styles/app.css、scripts/app.js
- app.css 只使用 var(--wg-*) 表达设计值
- app.js 中存在真实状态切换逻辑
- 每个字段有独立的校验逻辑
- 字段级错误有明确的视觉提示
- 提交按钮能触发所有字段校验
- 提交失败有错误反馈
- 修改字段后错误状态清除
- 失败后可重新提交
- 成功状态有明确反馈
