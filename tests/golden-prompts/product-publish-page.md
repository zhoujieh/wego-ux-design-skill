# 商品发布页回归测试

## 测试目标
验证 Skill 能否生成符合微购规范的商品发布页 Web 原型项目，包含多字段表单、图片上传、草稿保存、发布成功等完整链路。

## 输入提示词
使用 $wego-ux-design 生成一个微购商品发布页 Web 原型项目。用户是需要上架新商品的商家，需要填写商品名称、价格、库存、描述并上传商品图片，业务目标是提高商品上架效率。

## 期望页面类型
表单型

## 必须命中的组件
- Input
- Button
- Toast
- Dialog（草稿保存确认）

## 必须覆盖状态
- 初始状态
- 字段为空校验
- 价格格式错误
- 图片上传中
- 图片上传成功
- 图片上传失败
- 草稿保存确认
- 草稿保存成功
- 发布中
- 发布成功
- 发布失败
- 失败后可重新发布

## 必须使用 Token 类别
- color
- typography
- spacing
- radius
- size
- layout
- motion
- z-index

## 禁止问题
- 禁止输出单文件 HTML
- 禁止内联 style
- 禁止内联 script
- 禁止业务 CSS 硬编码 HEX / px / rgba
- 禁止只静态展示多个状态
- 禁止跳过失败恢复路径
- 禁止省略图片上传反馈

## 验收清单
- 输出目录包含 index.html、styles/tokens.css、styles/components.css、styles/app.css、scripts/app.js
- app.css 只使用 var(--wg-*) 表达设计值
- app.js 中存在真实状态切换逻辑
- 表单字段能触发校验
- 图片上传有加载中和成功/失败状态
- 草稿保存有确认和成功反馈
- 发布按钮能触发提交流程
- 失败状态能恢复
- 成功状态有明确结果反馈

## 信息层级要求

- 页面标题（NavBar 或顶部标题）为最高层级，使用 heading typography token
- 表单分组标题（如"基本信息"、"商品图片"、"价格库存"）为第二层级，视觉权重低于页面标题
- 表单字段标签为第三层级，与输入内容形成标签-值对比
- 辅助文字（如提示语、字数统计）为最低层级，使用 secondary/tertiary 文字颜色 token
- 操作按钮（发布）视觉权重最高，草稿保存按钮为次要操作

## 布局节奏要求

- 页面边距：左右 M2（var(--wg-spacing-24)），顶部 M0
- 表单分组之间：G2（var(--wg-spacing-24)），分组标题与首个字段之间 G1（var(--wg-spacing-16)）
- 同组字段之间：G1（var(--wg-spacing-16)）
- 字段标签与输入框之间：G1（var(--wg-spacing-8) 对应 --wg-spacing-8）
- 图片上传区域与说明文字之间：G1（var(--wg-spacing-8)）
- 底部操作区与内容区之间：G2（var(--wg-spacing-24)），操作按钮之间 G1（var(--wg-spacing-16)）

## 常见失败

- 层级扁平化：所有文字使用同一字号/颜色，无法区分标题、标签、辅助文字
- 间距不一致：分组间距与字段间距相同，缺乏节奏感
- 状态遗漏：只展示成功路径，缺少校验失败、上传失败、发布失败等错误状态
- 按钮权重不分：发布与草稿保存按钮视觉权重相同，用户无法快速识别主操作
- 图片上传无反馈：点击上传后无加载中状态，用户无法判断操作是否生效
- 表单分组缺失：所有字段平铺排列，无分组标题，信息扫描效率低
- 辅助文字喧宾夺主：提示语字号/颜色与标签相同，干扰主信息阅读

## 设计评分

100 分制，6 个维度：

| 维度 | 分值 | 说明 |
|------|------|------|
| 目标清晰度 | 20 | 页面是否清晰传达"发布商品"业务目标，主操作是否一目了然 |
| 信息层级 | 20 | 标题、分组、标签、辅助文字是否有明确的视觉层级区分 |
| 布局节奏 | 20 | 分组间距、字段间距、边距是否遵循 G1/G2/M0-M2 节奏规则 |
| 组件正确性 | 15 | 是否命中 Input/Button/Toast/Dialog 组件并正确使用 Canonical HTML/CSS |
| 状态完整性 | 15 | 是否覆盖全部 12 个必须状态，含失败恢复路径 |
| 微购风格 | 10 | 是否全部使用 var(--wg-*) Token，配色/圆角/间距是否符合微购视觉风格 |
