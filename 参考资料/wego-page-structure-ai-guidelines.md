# 微购页面结构与场景模式规范（AI 指导）

> 用途：指导 AI 在生成微购移动端页面时，先做页面结构与排版决策，再使用组件和 tokens。  
> 来源：基于 Figma 节点「✅ 页面结构与颜色」中罗列的页面与结构示例整理。  
> 适用范围：移动端 App / H5 页面结构、列表结构、表单结构、底部工具栏、卡片与通栏选择。  
> 不替代：组件规范、颜色 tokens、字体 tokens、间距 tokens。

---

## 1. 核心原则

AI 生成页面时，必须先判断页面结构，再拼装组件。页面不是组件的线性堆叠，而是由固定区域、滚动区域、内容分组、操作区域共同组成。

必须遵循以下原则：

1. **页面先分层，组件后填充**  
   页面必须先划分为：背景层、顶部导航层、二级导航/搜索层、主体滚动层、底部操作层、安全区。

2. **顶部导航必须固定**  
   页面滚动时，导航栏不能随内容滚走。搜索栏、筛选栏、二级 Tab 如果承担全局筛选，也应固定在导航栏下方。

3. **底部操作必须固定**  
   提交、批量操作、全选、结算、确认类操作不能放在滚动内容内部。底部工具栏必须固定在屏幕底部，并为滚动内容预留底部安全距离。

4. **一个页面只能有一个主操作**  
   同一操作区内只能有一个主按钮。多个并列操作时，只有最高优先级操作使用主按钮，其他使用次要按钮、文字按钮或灰底按钮。

5. **卡片必须对应业务实体**  
   卡片不是装饰容器。只有当每个卡片代表一个独立业务对象时，才允许使用卡片布局。

6. **同一分组必须保持同一对齐规则**  
   同一表单分组内，不允许一部分字段左对齐、一部分字段右对齐。字段标题、输入区、右侧操作必须保持统一列结构。

## 2. 标准页面骨架

移动端页面以 375 × 812 为基准参考。

### 2.1 基础结构

页面默认结构：

```text
Page
├── Background
├── NavBar / 顶部导航，固定
├── SecondaryBar / 搜索、筛选、Tab，可选，固定
├── ScrollView / 主体内容，可滚动
├── BottomBar / 底部操作区，可选，固定
└── SafeArea / iOS 安全区
```

### 2.2 常用高度

| 区域 | 常用高度 | 规则 |
|---|---:|---|
| 状态栏 + 导航栏 | 88 | 常规页面默认顶部高度 |
| 返回型导航 | 88 / 98 | 根据组件实际高度使用，必须固定 |
| 搜索区 | 56 | 通常位于 NavBar 下方，灰底承载 |
| 二级 Tabs | 48 | 可与搜索区组合为 96 高度 |
| ScrollView 起点 | 88 / 144 / 184 | 根据顶部固定区高度决定 |
| 底部普通工具栏 | 56 + 34 = 90 | 56 操作区 + 34 安全区 |
| 底部双按钮工具栏 | 80 + 34 = 114 | 80 按钮区 + 34 安全区 |
| 表格表头 | 44 | 多列横向表格使用 |
| 表格行高 | 48 | SKU / 多价格表格使用 |
| 常规 Cell | 56 | 设置项、表单项、普通列表 |
| 中等 Cell | 72 / 80 | 带头像、辅助信息、双行文本 |
| 复杂实体 Cell | 96 / 104 / 152+ | 商品、库存、规格组、员工信息 |

## 3. 背景与层级规则

### 3.1 页面背景

默认页面背景使用浅灰底，内容区域使用白色或白色卡片承载。

推荐结构：

```text
页面背景：浅灰
内容容器：白色
底部工具栏：半透明浅灰 / 毛玻璃 + 上描边
不支持毛玻璃时：浅灰底 + 上描边
```

### 3.2 底部工具栏背景

底部工具栏必须和内容区形成层级区分：

- 优先使用：浅灰半透明 + Background Blur + 顶部描边。
- 不支持毛玻璃时：使用浅灰实色 + 顶部描边。
- 工具栏覆盖滚动内容时，ScrollView 必须增加底部占位，避免最后一项被遮挡。

## 4. 页面类型与结构选择

### 4.1 表单编辑页

适用于：发布商品、编辑岗位、新增货位、设置权限、批发价设置等。

结构：

```text
NavBar 固定
ScrollView
  ├── FormGroup 1
  ├── FormGroup 2
  ├── 说明 / 辅助文案
  └── 底部占位
BottomBar 固定，可选
```

规则：

- 表单字段必须按业务分组，不允许所有字段无脑堆成一整块。
- 同一分组内字段行结构必须一致。
- 普通字段行高使用 56；说明多、输入复杂时可使用 64 / 72 / 80 / 94。
- 说明文案必须归属到对应分组下方，不要漂浮在页面中。
- 有提交动作时，提交按钮放在固定底部，不放入 ScrollView。

### 4.2 常规列表页

适用于：粉丝列表、员工列表、货位管理、仓库管理、批量删除颜色等。

结构：

```text
NavBar 固定
Search / Filter / Tabs 固定，可选
ScrollView
  ├── ListItem
  ├── ListItem
  ├── ListItem
  └── Footer / 底部占位
BottomBar 固定，可选
```

规则：

- 搜索、筛选、Tab 若影响整页数据，应固定在顶部，而不是随列表滚动。
- 普通列表用通栏 Cell，不使用卡片。
- 带头像、图片、状态、操作按钮时，行高使用 72 / 96。
- 批量模式必须出现底部选择工具栏，左侧展示选择状态，右侧展示操作按钮。

### 4.3 实体卡片列表页

适用于：盘点单、复杂商品、入库确认、库存相关业务对象。

结构：

```text
NavBar 固定
ScrollView
  ├── EntityCard
  │   ├── 主信息
  │   ├── 数据摘要
  │   ├── 图片/商品列表
  │   └── 操作区
  ├── EntityCard
  └── 底部占位
```

规则：

- 每张卡片必须代表一个真实业务实体，例如一张盘点单、一个商品组、一个入库确认项。
- 卡片内部至少有 2 层信息结构，例如标题 + 数据摘要 + 图片/明细 + 操作区。
- 卡片间距通常使用 8。
- 卡片宽度通常为 359，左右页面边距 8。
- 卡片内部内容边距通常为 16。
- 操作区应固定在卡片底部，不能散落在主内容中。

### 4.4 多列数据表格页

适用于：SKU 多价格、批量价格、库存、规格编码、条码、上架状态等多字段横向编辑场景。

结构：

```text
NavBar 固定
Content
  └── HorizontalTable
      ├── Header 44
      ├── Row 48
      ├── Row 48
      └── 批量填写 / 底部操作
```

规则：

- 当字段超过 5 列，且用户需要横向比较或批量编辑时，使用表格。
- 表格允许横向滚动，不要强行压缩到 375 宽度内。
- 表头高度 44，数据行高 48。
- 图片列可使用 56 宽；规格列可使用 108 宽；价格、库存、重量等字段可使用 88 宽；编码、条码等长字段可使用 200 宽。
- 表格底部可以提供“批量填写”等辅助操作，但不能替代主要保存动作。

### 4.5 搜索筛选页

适用于：颜色、规格、货位、库存盘点、粉丝客户等需要检索的数据页。

结构：

```text
NavBar 固定
SearchBar 固定
Tabs / Filter 固定，可选
ScrollView
  └── Results
```

规则：

- 搜索栏通常放在 56 高灰底容器中，左右边距 8，搜索框高度 36。
- 如果有筛选入口，放在搜索栏右侧，不要挤入导航标题区。
- 如果搜索 + Tabs 同时存在，组合区域高度通常为 96。
- 搜索区是功能控制区，不属于列表内容。

### 4.6 批量操作页

适用于：批量删除、批量选择、入库确认、库存盘点等。

结构：

```text
NavBar 固定
Search / Filter 固定，可选
ScrollView
  └── SelectableList
BottomBar 固定
  ├── Left: 选择状态 / 全选
  └── Right: 批量操作按钮
SafeArea
```

规则：

- 底部工具栏高度通常为 90。
- 左侧显示选择状态，例如“已选 / 未选择 / 全选”。
- 右侧最多放 2 个操作按钮。
- 删除、确认、完成等强动作只能有一个主按钮。
- 列表底部必须预留至少 90 + 8 的滚动安全距离。

## 5. 卡片 vs 通栏的判断规则

### 5.1 允许使用卡片的条件

满足以下任一条件，才适合使用卡片：

1. 页面主体由重复的业务实体组成。
2. 每个实体结构一致，只是填充值不同。
3. 每个实体内部有至少 2 层信息结构。
4. 每个实体需要独立操作区。
5. 用户需要按实体快速扫描、比较、处理。

典型场景：

- 盘点单列表
- 商品入库确认
- 复杂订单 / 商品组
- 仓库或货位实体列表（当每个实体有图标、状态、说明、操作时）

### 5.2 必须使用通栏的情况

以下情况统一使用通栏，不使用卡片：

1. 设置项列表。
2. 表单字段列表。
3. 简单文本列表。
4. 权限开关列表。
5. 字段编辑页。
6. 内容只是同一表单分组的一部分。
7. 卡片无法对应真实业务实体。

### 5.3 禁止错误用法

禁止把卡片当作视觉装饰容器使用。

错误示例：

- 一个卡片包住几个无关字段。
- 一个页面里每个表单分组都做成卡片，但卡片不对应业务对象。
- 简单设置列表用多张卡片分散承载。
- 为了“看起来有设计”而增加卡片层级。

## 6. 内容分组规则

### 6.1 分组依据

内容必须按任务和信息关系分组，而不是按组件类型分组。

优先分组方式：

1. 基础信息
2. 价格 / 库存 / 规格
3. 权限 / 可见范围
4. 物流 / 运费 / 重量
5. 高级设置
6. 说明 / 帮助信息
7. 危险操作

### 6.2 分组间距

- 分组之间默认使用 8 垂直间距。
- 搜索栏与列表之间通常不额外增加大间距。
- 标题 Cell 与内容列表之间应形成明确从属关系。
- 底部操作栏上方的内容必须预留安全占位。

### 6.3 分组内部

- 同一分组内部使用连续 Cell 或连续 Form Row。
- 同一分组内部不应混用多种对齐方式。
- 分割线应从内容起点开始，不必贯穿全屏，除非是强分割区域。

## 7. 表单排版规则

### 7.1 标准表单行

表单行结构：

```text
FormRow 56
├── Left Padding 16
├── Label / 标题区
├── Content / 输入区
├── Right Action / 图标、箭头、编辑按钮，可选
└── Divider / 分割线
```

### 7.2 对齐规则

- 同一分组内 Label 宽度必须统一。
- 输入区起点必须统一。
- 右侧操作必须统一靠右。
- 数字、价格、库存等编辑字段在同一分组内必须使用同一种对齐方式。
- 不允许在同一组里出现一部分输入靠左、一部分输入靠右。

### 7.3 高度规则

| 场景 | 建议高度 |
|---|---:|
| 单行输入 / 选择 | 56 |
| 带说明文案 | 64 / 72 |
| 多行输入 | 80 / 94 |
| 自定义内容块 | 根据内容自适应，但必须保持 8px 网格 |

## 8. 顶部区域规则

### 8.1 NavBar

- NavBar 必须固定在页面顶部。
- 标题居中。
- 左侧通常为返回或取消。
- 右侧最多放 1–2 个图标/文字操作。
- 右侧操作如果是页面主操作，需谨慎；保存/提交类更推荐放到底部或明确为唯一主操作。

### 8.2 二级导航 / 搜索 / 筛选

- 搜索栏高度通常为 56，搜索框高度 36，左右边距 8。
- Tabs 高度通常为 48。
- 搜索 + Tabs 组合时，整体高度通常为 96。
- 二级导航影响列表内容时，应固定在顶部。

---

## 9. 底部操作规则

### 9.1 普通底部按钮区

适用于提交、保存、确认、完成等。

结构：

```text
BottomBar 114
├── ButtonArea 80
│   ├── SecondaryButton 48，可选
│   └── PrimaryButton 48
└── SafeArea 34
```

规则：

- 单主操作：只放一个主按钮。
- 双操作：左次要，右主要。
- 两个操作同等重要时，两个都不能使用强主按钮，应使用次要按钮或弱按钮样式。
- 页面内不能同时出现导航右侧主操作和底部主操作，除非两者不是同一任务。

### 9.2 批量底部工具栏

适用于全选、批量删除、批量确认等。

结构：

```text
BottomBar 90
├── ActionArea 56
│   ├── Left: 选择状态
│   └── Right: 操作按钮
└── SafeArea 34
```

规则：

- 左侧展示选择状态，不放主要业务按钮。
- 右侧放操作按钮，最多 2 个。
- 危险操作不能和主确认操作视觉权重相同。

---

## 10. 列表与 Cell 规则

### 10.1 普通列表

- 使用 56 高 Cell。
- 左侧为标题或图标 + 标题。
- 右侧为值、箭头、开关、更多等。
- 适用于设置、权限、基础信息选择。

### 10.2 头像 / 用户列表

- 使用 72 高或更高 Cell。
- 左侧头像，右侧主标题 + 辅助说明。
- 操作按钮放右侧，避免打断主阅读路径。

### 10.3 商品 / 库存列表

- 使用 96 / 104 高复杂 Cell。
- 图片建议 64 或 72。
- 文本区按“标题 / 货号 / 价格 / 货位”等顺序排列。
- 计数器、选择框、状态按钮必须与主信息保持清晰层级。

### 10.4 实体卡片

- 卡片宽度常用 359，页面左右边距 8。
- 内边距常用 16。
- 操作区高度常用 56。
- 图片缩略图常用 32，连续横排时可用渐隐承接“更多”。

## 11. 空状态、少内容、多内容规则

AI 必须考虑同一页面的三种数据状态：

1. **空状态**：没有数据时，仍需保留搜索、添加入口、底部操作状态，不要让页面空白失衡。
2. **少内容**：内容不足一屏时，底部操作仍固定在底部，内容不要被强行拉满。
3. **多内容**：内容超过一屏时，顶部导航和底部工具栏固定，ScrollView 独立滚动。

多内容页面必须额外检查：

- NavBar 是否固定。
- Search / Tabs 是否固定。
- BottomBar 是否固定。
- ScrollView 是否预留底部占位。
- 最后一项是否会被工具栏遮挡。

## 12. AI 生成前必须输出的页面结构决策

AI 在生成页面前，必须先输出以下结构决策，再开始设计或编码：

```text
页面类型：表单编辑页 / 常规列表页 / 实体卡片列表页 / 表格编辑页 / 批量操作页 / 搜索筛选页
顶部结构：NavBar / NavBar + Search / NavBar + Search + Tabs
主体结构：通栏 Cell / FormGroup / EntityCard / HorizontalTable
是否使用卡片：是 / 否
卡片理由：对应的业务实体是什么
底部操作：无 / 单按钮 / 双按钮 / 批量工具栏
主操作：具体是哪一个
滚动规则：哪些区域固定，哪些区域滚动
底部安全距离：是否已预留
```

没有完成以上判断，不允许直接生成页面。

## 13. AI 自检清单

生成页面后必须逐项检查：

- [ ] 页面是否先完成结构分层，而不是组件堆叠。
- [ ] NavBar 是否固定在顶部。
- [ ] 搜索 / 筛选 / Tabs 是否根据场景固定。
- [ ] ScrollView 起点是否正确避开顶部固定区。
- [ ] 底部工具栏是否固定在底部。
- [ ] ScrollView 是否预留底部安全距离。
- [ ] 页面是否只有一个主操作。
- [ ] 多按钮时，是否只有最高优先级按钮使用主按钮。
- [ ] 卡片是否对应真实业务实体。
- [ ] 简单设置项、表单项是否使用通栏而不是卡片。
- [ ] 同一表单分组内字段对齐是否一致。
- [ ] 分组间距是否清晰统一。
- [ ] 多内容状态下，最后一项是否不会被底部栏遮挡。
- [ ] 空状态、少内容、多内容是否都能成立。

## 14. 禁止事项

AI 生成微购页面时，禁止以下行为：

1. 禁止直接把组件从上到下堆满页面。
2. 禁止页面滚动时导航栏消失。
3. 禁止把底部操作栏放进 ScrollView。
4. 禁止一个页面出现两个主按钮。
5. 禁止多个同级按钮都使用主按钮样式。
6. 禁止卡片不对应业务实体。
7. 禁止简单表单字段乱用卡片包装。
8. 禁止同一表单分组内字段左右对齐混乱。
9. 禁止搜索栏、筛选栏、Tabs 和列表内容混成同一滚动层。
10. 禁止底部工具栏遮挡最后一条内容。
11. 禁止为了视觉丰富而增加无意义容器。
12. 禁止用颜色、阴影、卡片弥补信息架构混乱。

## 15. 可直接加入 AI 规则的执行提示词

生成微购页面时，必须先进行页面结构判断。不要直接拼组件。

你必须按以下顺序执行：

1. 判断页面类型：表单、列表、实体卡片、表格、批量操作、搜索筛选。
2. 确定固定区域：NavBar、Search、Tabs、BottomBar。
3. 确定滚动区域：只有主体内容进入 ScrollView。
4. 判断主体使用通栏、卡片还是表格。
5. 若使用卡片，必须说明卡片对应的业务实体；不能说明则改为通栏。
6. 判断底部操作优先级：一个页面只能有一个主操作。
7. 为底部固定栏预留滚动安全距离。
8. 检查同一分组内字段对齐是否一致。
9. 最后再使用组件和 tokens 完成页面。

输出页面时，必须同时给出：页面结构、固定区域、滚动区域、内容分组、主操作、卡片/通栏选择理由。
