# 06 Output Format

> 微购 Design System Skill / rules
> Version 2.0
> 本文档定义被 `SKILL.md` 选中任务的具体输出格式。

---

# 1. 基础要求

任务类型和输出形式以 `SKILL.md` 为准。本文档不得把生成任务改成方案说明，也不得为当前任务增加新的输出类型。

AI 输出必须：

- 直接
- 可执行
- 有结构
- 不空泛
- 不输出无意义解释
- 不覆盖微购已有规范

---

# 2. 界面生成输出格式

页面或组件原型生成任务交付：

```text
一个完整、可运行、包含全部交互文件和所需资源的项目目录
```

最低要求：

```text
index.html
styles/tokens.css
styles/components.css
styles/app.css
scripts/app.js
assets/（按实际需要）
```

- HTML、CSS、JavaScript 分离。已注册组件的 Canonical CSS 写入 `styles/components.css`，页面级业务样式写入 `styles/app.css`。
- 所有本地引用必须在交付目录内可解析。
- 使用框架或构建工具时，包含依赖清单、入口文件和启动命令。
- 不创建任务不需要的空目录或占位资源。
- 默认不输出页面目标、信息层级、Token 清单或自查过程。

当用户明确要求设计说明时，可额外补充：

```text
页面目标：
布局与信息层级：
关键组件与状态：
```

说明必须简短，且不能替代项目文件。

最终回复必须说明：

```text
项目入口：
启动方式：
关键交互：
验证结果：
```

# 6. 代码输出规则

输出代码时：

- Token 定义文件可以包含原始设计值
- 业务样式不硬编码颜色、未定义间距或未定义圆角
- 不使用随机 z-index
- 样式值必须能映射到 `wg.*`
- 缺少 Token 时必须注释说明
- HTML 不内联业务 CSS
- HTML 不内联业务 JavaScript
- 资源路径必须与交付目录一致

---

# 7. 简短输出规则

当用户明确要求：

```text
直接输出
不要解释
给我文件
可下载
```

生成任务只输出项目目录链接和必要运行信息；审查、Token 检查和规范问答仍按 `SKILL.md` 对应的任务形式输出结果。
