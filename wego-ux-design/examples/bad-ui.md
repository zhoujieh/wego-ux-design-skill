# 反例：订单提交页

## 错误表现

```text
页面顶部使用大面积渐变绿色背景
每个商品卡片都有强阴影
页面中同时出现多个主按钮
金额、优惠、状态全部使用红色
间距使用 10px / 14px / 20px
圆角使用 20px
按钮使用 z-index: 9999
错误提示只写“失败”
```

---

# 问题分析

## P0

问题：

```text
主操作不清晰，多个按钮抢焦点。
```

原因：

```text
页面没有建立唯一核心视觉重心。
```

修改建议：

```text
保留一个主操作，其余操作降级为次操作或文本操作。
```

---

## P1

问题：

```text
大量渐变、阴影和红色破坏微购克制风格。
```

原因：

```text
用装饰代替信息层级。
```

修改建议：

```text
移除渐变和重阴影，通过标题、金额、布局和按钮层级建立重点。
```

---

## P1

问题：

```text
使用未定义间距和圆角。
```

原因：

```text
没有遵守 Token。
```

修改建议：

```text
间距改为 wg.spacing.*
圆角改为 wg.radius.*
```

---

## P2

问题：

```text
错误提示不可恢复。
```

原因：

```text
“失败”没有说明原因和下一步。
```

修改建议：

```text
改为“提交失败，请重试”。
```

---

# 禁止复用的错误方式

禁止：

- 大面积渐变
- 强阴影堆叠
- 多个主按钮
- 高饱和色滥用
- 非标准间距
- 非标准圆角
- 魔法 z-index
- 不可恢复错误文案

# 审查输出反例

以下内容来自旧审查示例，保留用于识别典型问题和输出格式。

## 结论

```text
当前界面主要问题是信息密度不足、主次层级不清、Token 使用不统一。
```

## P0

问题：

```text
商品列表中主操作和次操作视觉权重相同，用户难以判断下一步。
```

原因：

```text
同一区域存在多个强焦点。
```

修改建议：

```text
只保留一个主操作，其余操作降级为文本操作或次操作。
```

## P1

问题：

```text
列表间距过大，影响高密度商品浏览效率。
```

原因：

```text
高密度列表没有使用 M0 / G1。
```

修改建议：

```text
页面布局改为 M0，连续商品信息使用 G1。
```

问题：

```text
价格、标签、状态同时使用高饱和色。
```

原因：

```text
颜色语义混乱。
```

修改建议：

```text
价格使用 wg.color.text.promotion；状态使用对应的 wg.color.state.* Token；普通标签降级为中性色。
```

## P2

问题：

```text
辅助说明字号过大。
```

原因：

```text
辅助信息使用了正文层级。
```

修改建议：

```text
辅助说明改为 F12。
```

## Token 检查样例

合规：

```text
使用了 wg.color.text.primary
使用了 wg.radius.md
```

不合规：

<!-- token-lint: allow-hardcoded reason=review-input -->

```text
margin: 10px
font-size: 15px
border-radius: 20px
```

## 交互链路样例

已覆盖：

```text
商品点击进入详情
加载失败后可以重试
```

缺失或断链：

```text
批量操作提交后没有处理中状态
删除失败后没有恢复路径
```
