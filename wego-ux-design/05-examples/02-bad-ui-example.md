# 反例：订单提交页

## 错误表现

```text
页面顶部使用大面积渐变绿色背景
每个商品卡片都有强阴影
页面中同时出现多个主按钮
金额、优惠、状态全部使用红色
间距使用 10px / 14px / 20px
圆角使用 20px
按钮使用 z-index: 9999
错误提示只写“失败”
```

---

# 问题分析

## P0

问题：

```text
主操作不清晰，多个按钮抢焦点。
```

原因：

```text
页面没有建立唯一核心视觉重心。
```

修改建议：

```text
保留一个主操作，其余操作降级为次操作或文本操作。
```

---

## P1

问题：

```text
大量渐变、阴影和红色破坏微购克制风格。
```

原因：

```text
用装饰代替信息层级。
```

修改建议：

```text
移除渐变和重阴影，通过标题、金额、布局和按钮层级建立重点。
```

---

## P1

问题：

```text
使用未定义间距和圆角。
```

原因：

```text
没有遵守 Token。
```

修改建议：

```text
间距改为 wg.spacing.*
圆角改为 wg.radius.*
```

---

## P2

问题：

```text
错误提示不可恢复。
```

原因：

```text
“失败”没有说明原因和下一步。
```

修改建议：

```text
改为“提交失败，请重试”。
```

---

# 禁止复用的错误方式

禁止：

- 大面积渐变
- 强阴影堆叠
- 多个主按钮
- 高饱和色滥用
- 非标准间距
- 非标准圆角
- 魔法 z-index
- 不可恢复错误文案
