# 05 Forbidden Rules

> 微购 Design System Skill / 04-ai-rules  
> Version 2.0  
> 本文档定义 AI 在微购设计任务中的禁止事项。

---

# 1. 视觉禁止事项

禁止：

- 滥用渐变
- 滥用阴影
- 滥用毛玻璃
- 滥用高饱和色
- 滥用图标
- 滥用卡片嵌套
- 滥用圆角
- 用装饰替代信息层级
- 为了“高级感”牺牲清晰度

---

# 2. Token 禁止事项

禁止：

- 直接写 HEX
- 直接写 RGBA
- 直接写未定义 px
- 新增未定义字号
- 新增未定义间距
- 新增未定义圆角
- 使用随机 z-index
- 创建单页面专属 Token

---

# 3. 图标禁止事项

禁止：

- 在 iconfont 中存在所需图标时使用 SVG 内联或图片替代
- 硬编码图标尺寸（必须使用 `wg.size.*` Token）
- 硬编码图标颜色（必须使用 `wg.color.*` Token 或继承父元素）
- 自行创造 iconfont 中不存在的图标样式
- 使用 emoji 或图片替代功能图标
- 不查 iconfont.json 直接使用 SVG 内联图标

---

# 4. 组件禁止事项

禁止：

- 将页面级临时结构声明为正式组件规则
- 同一区域多个主按钮
- 弹窗承载长流程
- Toast 承载关键决策
- 图标替代不清晰的文字
- 单页面临时创造组件变体
- 组件状态不完整

---

# 5. 布局禁止事项

禁止：

- 页面没有主次结构
- 多个视觉中心同时存在
- 高密度页面使用过大留白
- 低密度页面过度压缩
- 无规则混用 M / G 模式
- 通过临时 padding 伪造页面布局

---

# 6. 文案禁止事项

禁止：

- 文案模糊
- 只有错误没有恢复建议
- 空状态没有说明
- 危险操作没有说明后果
- 使用恐吓式文案
- 使用技术术语替代用户语言

---

# 7. AI 行为禁止事项

禁止：

- 盲目推理输出
- 忽略用户已有规范
- 覆盖微购设计语言
- 输出不可执行建议
- 不读规则直接生成
- 用通用 UI 规则替代微购规则
- 把讨论过程写进正式规范文档
- 用静态状态陈列代替关键交互链路
- 将业务样式和交互脚本内联进 HTML
