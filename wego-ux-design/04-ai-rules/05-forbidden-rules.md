# 05 Forbidden Rules

> 微购 Design System Skill / 04-ai-rules  
> Version 2.0  
> 本文档定义 AI 在微购设计任务中的禁止事项。

---

# 1. 视觉禁止事项

禁止：

- 滥用渐变
- 滥用阴影
- 滥用毛玻璃
- 滥用高饱和色
- 滥用图标
- 滥用卡片嵌套
- 滥用圆角
- 用装饰替代信息层级
- 为了“高级感”牺牲清晰度

---

# 2. Token 禁止事项

禁止：

- 直接写 HEX
- 直接写 RGBA
- 直接写未定义 px
- 新增未定义字号
- 新增未定义间距
- 新增未定义圆角
- 使用随机 z-index
- 创建单页面专属 Token

---

# 3. 图标禁止事项

禁止：

- 在 iconfont 中存在所需图标时使用 SVG 内联或图片替代
- 硬编码图标尺寸（必须使用 `wg.size.*` Token）
- 硬编码图标颜色（必须使用 `wg.color.*` Token 或继承父元素）
- 自行创造 iconfont 中不存在的图标样式
- 使用 emoji 或图片替代功能图标
- 不查 iconfont.json 直接使用 SVG 内联图标
- iconfont 图标元素未设置 `line-height: 1` 导致垂直偏移

---

# 4. 组件禁止事项

禁止：

- 将页面级临时结构声明为正式组件规则
- 同一区域多个主按钮
- 弹窗承载长流程
- Toast 承载关键决策
- 图标替代不清晰的文字
- 单页面临时创造组件变体
- 组件状态不完整
- 同一业务动作在页面中出现多个 primary / strong 表达
- 跨区域重复主操作都使用强按钮
- 卡片不对应真实业务实体
- 把简单表单分组包装成无意义卡片
- 设置项、字段项、简单文本列表滥用卡片
- 页面内出现多个同级主按钮
- 未完成页面结构判断就直接拼组件
- 把页面做成组件堆叠
- Dialog / Sheet / Popover 内容无结构、无间距、无点击态
- 组件缺少必要交互状态（default / hover / active / disabled / error / loading）
- 组件 CSS 存在非法单位或不可生效代码

---

# 5. 布局禁止事项

禁止：

- 页面没有主次结构
- 多个视觉中心同时存在
- 高密度页面使用过大留白
- 低密度页面过度压缩
- 无规则混用 M / G 模式
- 通过临时 padding 伪造页面布局
- 同一个 form group 内左右对齐混用
- 导航区被普通内容滚动带走
- 搜索栏、筛选栏、Tabs 和列表内容混成同一滚动层
- 底部工具栏进入 ScrollView
- 底部工具栏遮挡最后一条内容
- 多列表格强行压缩到 375px 宽度内
- 大屏下业务页面内容超过最大宽度约束

---

# 6. 文案禁止事项

禁止：

- 文案模糊
- 只有错误没有恢复建议
- 空状态没有说明
- 危险操作没有说明后果
- 使用恐吓式文案
- 使用技术术语替代用户语言
- 空 spinner 代替明确 loading 状态
- 错误状态在用户修正后仍残留

---

# 7. AI 行为禁止事项

禁止：

- 盲目推理输出
- 忽略用户已有规范
- 覆盖微购设计语言
- 输出不可执行建议
- 不读规则直接生成
- 用通用 UI 规则替代微购规则
- 把讨论过程写进正式规范文档
- 用静态状态陈列代替关键交互链路
- 将业务样式和交互脚本内联进 HTML

---

# 8. 状态与反馈禁止事项

禁止：

- Loading 没有任务语义（空转圈、无文案全屏 loading）
- Loading 完成后页面没有状态变化
- 提交类 Loading 不防重复点击
- 错误修正后不清除 error class / error 文案 / aria-invalid
- 可修正的错误没有恢复路径
- 同一业务动作在 NavBar、底部操作区、内容区同时使用 strong
