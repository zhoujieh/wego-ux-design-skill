# 05 Requirement Confirmation Rules

> 微购 Design System Skill / 04-ai-rules
> Version 1.0
> 本文档定义 AI 在执行设计任务前的需求确认流程。

---

# 1. 目标

AI 在进入页面类型判断和界面生成之前，必须先确认三个核心维度。缺少任一维度，设计方向就可能偏离真实诉求。

```text
用户任务 — 用户要完成什么？完成标准是什么？
业务目标 — 做成什么样算成功？核心衡量标准？
使用场景 — 谁在什么情况下用？进入时的状态和预期？
```

---

# 2. 输入分类

根据用户输入的信息密度，将需求分为三类，决定后续提问策略：

| 信号 | 类型 | 提问策略 |
|------|------|---------|
| 包含截图、Figma 链接、线框图描述、具体布局说明 | 草图型 | 少问，只补缺口 |
| 只有一句话需求、概念性描述 | 模糊型 | 逐个深挖三个维度 |
| 包含"参考""像 XX 那样""照着做"、竞品截图或链接 | 参考型 | 分析差异，确认借鉴范围 |
| 上述混合 | 混合型 | 按优先级处理：参考分析 → 草图确认 → 模糊深挖 |

---

# 3. 三个维度的确认

## 3.1 用户任务

用户来到这个页面，要完成什么事？完成的标准是什么？

这是所有设计决策的根。不知道用户任务，就会做出"功能齐全但没人用"的页面。

**提问方式：**

- 草图型：复述确认。"我理解用户在这个页面主要要完成 [任务]，对吗？"
- 模糊型：5Why 追问本质。不停留在表面需求，追问到用户真正的目的。
- 参考型：对比差异。"参考产品的用户任务是 [X]，我们的用户任务和它一样吗？"

**确认标准：** 能用一句话说清"用户要完成什么"，且不是功能描述而是任务描述。

```text
✓ "用户要快速找到想买的商品并加入购物车"
✗ "做一个商品列表页"
```

## 3.2 业务目标

这个页面做成什么样算成功？最核心的一个衡量标准是什么？

业务目标决定设计方向。不知道业务目标，就会做"好看但不对"的页面。

**提问方式：**

- 草图型：直接问。"这个页面做成什么样算成功？"
- 模糊型：用选择题引导收敛。"这个页面最核心的目标是：A) 让用户快速完成操作 B) 让用户发现更多内容 C) 让用户安心完成重要操作？"
- 参考型：对比差异。"参考产品的业务目标是 [X]，我们的目标一样吗？"

**确认标准：** 能明确说出一个优先级最高的业务目标。

```text
✓ "提高下单转化率"
✓ "降低用户操作成本"
✗ "做一个好用的页面"
```

## 3.3 使用场景

谁在什么情况下用这个页面？进入页面时的状态和预期是什么？

使用场景决定设计方式。不知道使用场景，就会做"通用但不对味"的设计。

**提问方式：**

- 草图型：问"给谁用的"和"在什么情况下进入这个页面"。
- 模糊型：用 5W1H 还原场景。什么类型的用户（Who），在什么情况下（When/Where），因为什么触发（What），想做什么（How）。
- 参考型：对比差异。"参考产品的用户群体是 [X]，我们的用户和它一样吗？"

**确认标准：** 能描述出具体的用户画像和使用情境。

```text
✓ "中年用户在通勤时快速浏览推荐商品"
✓ "新用户首次进入，需要引导完成注册"
✗ "所有用户"
```

# 4. Figma 集成

当用户输入中包含 Figma 链接时：

1. 检查 Figma MCP 是否可用
2. 可用 → 调用 Figma MCP 读取设计稿，提取布局结构、组件层级、交互标注，作为草图型输入处理
3. 不可用 → 提示用户："Figma MCP 未连接，请提供设计稿截图或文字描述"
4. 没有链接时跳过，不影响流程

---

# 5. 确认输出

三个维度都有明确答案后，AI 输出需求理解确认：

```text
我理解这个页面是：
[谁] 在 [什么情境] 下，要 [完成什么任务]，
业务目标是 [什么]。
对吗？
```

用户确认后 → 进入页面类型判断。

用户修正 → 针对修正的维度重新确认，不需要全部重来。

---

# 6. 确认完成标准

三个维度都有明确答案，且满足：

- 用户任务：是任务描述，不是功能描述
- 业务目标：有一个明确的优先级最高的目标
- 使用场景：能描述出具体的用户和情境，不是"所有用户"

未满足标准时，继续提问，不得跳过进入设计阶段。

---

# 7. 简短模式

当用户要求"直接输出""不要解释"时，进入简短模式。

**简短模式不跳过确认输出，只跳过等待用户确认。**

- 如果用户输入已包含三个维度的信息：AI 必须输出确认语句（第 5 节格式），但不需要等待用户确认，可立即进入阶段二。
- 如果缺少关键维度：简短确认缺失部分，不超过一个问题，获得回答后输出确认语句再进入阶段二。

**无论哪种模式，确认语句必须输出。** 确认语句的作用是让 AI 显式表达对需求的理解，让用户在后续交互中有机会纠偏。
